## Codebase Patterns
- Scraper usa requests.Session com headers HEADERS definidos no topo do módulo
- Parsing HTML usa BeautifulSoup com seletor div.resultadoItem para cada card
- Payload do TJSC usa campos hdnTotalPaginas e hdnPaginaAtual para paginação
- Variáveis de ambiente GCP: GCP_PROJECT_ID, BQ_DATASET_ID, BQ_TABLE_ID
- Modelo de embedding: intfloat/multilingual-e5-base com prefixo "passage: {text}"
- BigQuery insert: chunks de 256 com retry 3x (padrão de reindex.py no app principal)
- ID de registro: SHA-256 de "{processo}|{data_julgamento}"
- checkpoint.json fica na raiz do repo (não dentro de screping/)
- HTML cards usam div.resLabel e div.resValue (não span) para campos
- extract_processo usa a.numero-processo como prioridade, fallback para campo PROCESSO (split()[0])
- Usar uv para criar .venv e instalar deps: `uv venv .venv && uv pip install ... --python .venv/bin/python`
- scrape_month aceita failed_pages: list | None = None para coletar falhas sem alterar tipo de retorno
- Retry usa time.sleep(2**attempt) onde attempt ∈ {0,1} (apenas antes da 2ª e 3ª tentativa)
- 3 páginas consecutivas sem resultado (ou com falha) encerram o mês — failed pages contam como consecutivas
- patch("screping.scraper.fetch_page") + patch("time.sleep") é o padrão para testes de scrape_month
